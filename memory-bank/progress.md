#### Phase 5 — Real-time collaboration — COMPLETE
- **Transport (Architecture Decision 1 — SSE)**: a new in-process tier under `src/realtime/`. Chosen over WebSocket because it (a) mounts as an ordinary `/api/v1` route → preserves the `createApp()` supertest seam (R6), (b) rides the existing Vite `/api/v1` HTTP proxy with NO `ws:true` change → dev/prod parity (C2), (c) gets native `EventSource` auto-reconnect for free. **Zero new runtime deps** (SSE + `EventSource` are platform built-ins).
- **`src/realtime/broadcaster.ts`**: HTTP-ignorant pub/sub hub — `Map<boardId, Set<Subscriber>>` with `subscribe`/`unsubscribe`/`publish`/`connectionCount` (+ test-only `clear`). Empty channel Sets are pruned; `publish` guards each `send` so one dead sink can't break the fan-out; `connectionCount(boardId?)` is the ready accessor for a deferred `banyanboard_realtime_connections` gauge (metrics still deferred).
- **`src/realtime/events.ts`**: the full-entity board-scoped event contract (Decision 2) — `card:created|updated|deleted` (carry full `Card`) + `board:updated` (carries full `Board`), envelope with `boardId`/`originId`/`emittedAt`/`traceId`. Mirrored on the frontend in `client/src/api/types.ts`.
- **`src/realtime/eventsRouter.ts`**: `GET /api/v1/boards/:boardId/events` (`text/event-stream`), mounted in `src/routes/index.ts` AFTER cards. Validates `:boardId` (400 on bad id), 404s when `REALTIME_ENABLED=false` (GP5-safe — no "disabled" detail leaked), registers the `res` as a subscriber, writes `id:`/`event:`/`data:` frames, sends keep-alive comment frames every `REALTIME_KEEPALIVE_MS`, and on `req` close clears the timer + unsubscribes (no dead-subscriber leak). Lifecycle logged via the already-bound `req.log` (open/published/write-failed/close, all traceId-correlated; zero `console.*`).
- **`src/realtime/notify.ts`**: the mutation→broadcast bridge the routers call AFTER `res.json()` — builds the event (stamping `originId` from the `X-Client-Id` header + `traceId`), gated by `config.realtimeEnabled`, wrapped so a broadcast failure logs at warn and can NEVER fail the already-responded HTTP mutation (NFR1, fire-and-forget).
- **Mutation hooks**: `cards.ts` POST→`card:created`, PATCH→`card:updated` (covers edit AND drag status change), DELETE→`card:deleted` (delete reordered to findById-then-remove so the event carries the full entity; 404-on-absent behavior unchanged); `boards.ts` PATCH→`board:updated`. Board CREATE intentionally does not broadcast (no board-scoped channel exists for a brand-new board).
- **Config (Architecture Decision 5 / GP1)**: `REALTIME_ENABLED` (parseBool, default `true`) + `REALTIME_KEEPALIVE_MS` (new `parsePositiveInt`, default `15000`) added to the single frozen `src/config/env.ts` source, fail-fast.
- **Echo de-dup (Decision 3 / R5 — the critical interaction)**: writes send the per-tab id as the `X-Client-Id` *header* → echoed into the event `originId`; the SSE GET carries it as `?clientId=` *query param* (EventSource can't set headers) for connection logs only. `client/src/realtime/useRealtimeBoard.ts` (native `EventSource`, handlers kept in a ref so the stream isn't reopened per render, auto-reconnect, closes on unmount) **drops any event whose `originId` equals this tab's id** — so an optimistic move is never double-applied and the user never sees their own change flash.
- **Frontend wiring**: `BoardViewPage` calls `useRealtimeBoard(id, getClientId(), {onCardEvent, onBoardEvent})`; remote `card:*` events swap the full entity into state by id (create appends, update replaces, delete removes) and add the id to a `highlightedCardIds` set cleared after 650 ms; `board:updated` updates the heading in place (no flash — self-evident). The set threads `KanbanBoard → Column → CardItem` as a `recentlyUpdated` prop driving a CSS `recentlyUpdated` highlight-fade (600 ms, existing tokens, `prefers-reduced-motion` honored, no toast — UI/UX Spec 7).
- **Tests (+9 frontend, +12 backend)**: backend — `broadcaster.test.ts` 5 (delivery / board-scope isolation / no-subscriber no-op / unsubscribe-shrinks-count / per-board+total count), `eventsRouter.test.ts` 3 (real-server stream connect + SSE headers + frame delivery + disconnect cleanup / `REALTIME_ENABLED=false`→404 no-leak / bad id→400 no subscription), `mutationBroadcast.test.ts` 4 (POST→card:created w/ originId from X-Client-Id / PATCH→card:updated status / PATCH board→board:updated / board-scope no cross-board leak). Frontend — `useRealtimeBoard.test.ts` 5 (opens scoped EventSource / routes card:created / routes board:updated / drops own-origin echo [R5] / closes on unmount), CardItem +1 (recentlyUpdated highlight class), BoardViewPage +3 (remote card:created appears in its column / remote card:updated moves card columns / remote board:updated renames heading). **Backend Jest 150/150** (was 138; +12), **client Vitest 118/118** (was 109; +9); backend `tsc` + client `tsc -b`+`vite build` clean (74 modules, 232 KB / 75 KB gzip). Strict `tsc` is the lint gate.
